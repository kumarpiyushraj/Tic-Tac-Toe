<br/>

**Tic-Tac-Toe** is a classic strategy game reimagined for Android. Built entirely in **Java** with XML layouts in Android Studio, this app delivers a polished, feature-rich gaming experience that goes well beyond the basic grid. From a **Lottie-animated splash screen** to a **position-aware AI opponent**, every detail has been thoughtfully crafted.

Whether you're challenging the computer or battling a friend on the same device, the app handles every scenario — victories, draws, score tracking, and mid-game resets — with smooth animations and satisfying sound feedback.

<br/>

<div align="center">

| &nbsp; | Why This App? | What It Delivers |
|:---:|:---|:---|
| 🎮 | **Dual Game Modes** | Player vs Computer and Player vs Player on the same device |
| 🤖 | **Smart AI** | Position-aware computer opponent with strategic move prediction |
| 🏆 | **Win Line Highlighting** | 8 distinct win-line ImageViews animate over the board on victory |
| 🔊 | **Rich Audio** | 4 dedicated MediaPlayer instances for clicks, wins, draws, and board taps |
| 🎨 | **Custom UI Components** | Custom Snackbars, custom Toasts, themed AlertDialogs, and marquee player names |
| ✏️ | **Full Personalization** | Choose your name (up to 12 chars), pick your symbol (X or O), select your mode |

</div>

### 👥 3. Two-Player (PvP) Mode

The `GameBoard` activity handles full local two-player gameplay with real-time turn management and win detection.

<div align="center">

| Feature | Detail |
|:---|:---|
| 🔄 **Turn System** | `mCount % 2` determines whose turn it is; `tv1` banner updates background and text each turn |
| 🎨 **Symbol Colors** | O is rendered in blue (`#3e59c7`); X is rendered in red (`#f2071b`) on the board |
| 🏅 **Symbol Badges** | `btn1` and `btn2` TextViews show each player's symbol with colored backgrounds |
| 🖼️ **Player Icons** | `player` and `robot_player` ImageViews show colored avatars next to score labels |
| 📊 **Live Score** | `p1s` and `p2s` static counters increment on win; displayed as `Score - N` in `sc1`/`sc2` |
| 🔄 **Reset Board** | Clears all 9 cells, hides win lines, resets `mCount` to original `cH`, re-enables all tiles |
| 🗑️ **Reset Score** | Sets both `p1s` and `p2s` to 0 — guarded by custom Snackbar if scores are already zero |
| 📢 **Custom Snackbar** | Two pre-built `Snackbar` instances with inflated custom layout, OK button, and transparent background |
| 🔊 **Sound Effects** | `music5` (button), `music6` (winner), `music7` (game_over), `music8` (board_click) |

</div>

```java
// GameBoard.java — Symbol placement with turn tracking
public void setSymbolOnBoard(TextView textview) {
    mVar = (mCount % 2);
    mCount++;
    if (mVar == 0) {
        textview.setText("O");
        tv1.setBackgroundResource(R.drawable.turn_play_red);
        tv1.setText(R.string.x_turn);   // "X's Turn" shown next
        textview.setTextColor(0xFF3e59c7);
        textview.setEnabled(false);
    } else {
        textview.setText("X");
        tv1.setBackgroundResource(R.drawable.turn_play_blue);
        tv1.setText(R.string.o_turn);   // "O's Turn" shown next
        textview.setTextColor(0xFFf2071b);
        textview.setEnabled(false);
    }
}
```

<a name="4-single-player-pvc-mode--ai"></a>

### 🤖 4. Single-Player (PvC) Mode & AI

`GameBoardSinglePlayer` extends the PvP logic with a full AI turn system built on top of position-aware move prediction.

<div align="center">

```mermaid
sequenceDiagram
    participant User
    participant Board as GameBoardSinglePlayer
    participant AI as Computer Logic
    participant Handler as Handler (500ms delay)

    User->>Board: Tap a cell
    Board->>Board: setSymbolOnBoard(cell, byUser=true)
    Board->>Board: Disable all empty cells (block input)
    Board->>Board: isFromUser=false, isComputer=true
    Board->>Board: Check win / draw conditions

    alt Game not ended
        Board->>AI: isFirstMove? → setSymbolComputerRandom()
        Board->>AI: else → callMethod(lastPos) → position-aware method
        AI-->>Handler: postDelayed 500ms
        Handler->>Board: setSymbolComputer(targetCell)
        Board->>Board: setSymbolOnBoard(targetCell, byUser=false)
        Board->>Board: performClick() on targetCell
        Board->>Board: Re-enable remaining empty cells
        Board->>Board: isFromUser=true
    end
```

</div>

**AI Strategy — Two-Phase Decision Engine:**

<div align="center">

| Phase | Method | Behaviour |
|:---:|:---:|:---|
| **First Move** | `setSymbolComputerRandom()` | Picks a random empty cell using `java.util.Random` |
| **Subsequent Moves** | `callMethod(lastPos)` | Dispatches to position-specific method based on user's last move |
| **Fallback** | `tOne()` / `tThree()` / others | Tries alternative line completions if primary check returns null |
| **Final Fallback** | `setSymbolComputerRandom()` | Falls back to random if no strategic move is found |

</div>

```java
// GameBoardSinglePlayer.java — Position-aware AI dispatch
@Nullable
private TextView callMethod(int x) {
    if (x == 0) return tOne();    // User played T1 → check T1's lines
    else if (x == 1) return tTwo();
    else if (x == 2) return tThree();
    else if (x == 3) return tFour();
    else if (x == 4) return tFive();  // Centre — checks all 4 lines through T5
    else if (x == 5) return tSix();
    else if (x == 6) return tSeven();
    else if (x == 7) return tEight();
    else if (x == 8) return tNine();
    else return null;
}
```

**Position Method Logic (example — `tFive()`):**

```java
// T5 (centre) is the most strategically connected cell — checks all 4 winning lines
@Nullable
private TextView tFive() {
    if (T5.getText().equals(T1.getText()) && T9.getText().equals("")) return T9;  // diagonal
    if (T5.getText().equals(T9.getText()) && T1.getText().equals("")) return T1;  // diagonal
    if (T5.getText().equals(T3.getText()) && T7.getText().equals("")) return T7;  // diagonal
    if (T5.getText().equals(T7.getText()) && T3.getText().equals("")) return T3;  // diagonal
    if (T5.getText().equals(T4.getText()) && T6.getText().equals("")) return T6;  // row
    if (T5.getText().equals(T6.getText()) && T4.getText().equals("")) return T4;  // row
    if (T5.getText().equals(T2.getText()) && T8.getText().equals("")) return T8;  // column
    if (T5.getText().equals(T8.getText()) && T2.getText().equals("")) return T2;  // column
    return null;
}
```

**Additional PvC-Specific Flags:**

<div align="center">

| Flag | Type | Purpose |
|:---|:---:|:---|
| `isFromUser` | boolean | Prevents double-input while AI is thinking |
| `isComputer` | boolean | Gates AI move execution after user input |
| `gameHasEnded` | boolean | Stops both user and AI input after terminal state |
| `isFirstMove` | boolean | Routes first AI response to random picker |
| `lastPos` | int | Index of user's last cell tap (0–8) for AI dispatch |
| `handler` | Handler | Posts AI move with 500 ms delay for natural feel |
| `names[]` | TextView[] | Array of 9 board cells for index-based AI placement |

</div>

### 🏆 5. Win Detection & Draw System

Both `GameBoard` and `GameBoardSinglePlayer` share the same win detection logic through `cal1()` and `cal2()`.

<div align="center">

```mermaid
graph TD
    A[Cell Tapped] --> B[setSymbolOnBoard]
    B --> C[cal1 — Check 3 cells match]
    C -->|Match found| D{Which symbol won?}
    D -->|Player 1's symbol| E[Show win banner — win_red background<br/>Increment p1s score<br/>Disable remaining cells<br/>Show win-line ImageView]
    D -->|Player 2's symbol| F[Show win banner — win_blue background<br/>Increment p2s score<br/>Disable remaining cells<br/>Show win-line ImageView]
    C -->|No match| G[cal2 — isBoardFilledUp?]
    G -->|All 9 cells = No| H[Show Draw banner<br/>game_draw background<br/>Hide turn indicator]
    G -->|Empty cell remains| I[Continue — next player's turn]
```

</div>

```java
// Shared win detection — handles both symbol colors and both player score slots
public boolean cal1(TextView a, TextView b, TextView c, TextView result,
                    TextView score1, TextView score2, TextView turnView,
                    TextView symbolBadge, String p1WinMsg, String p2WinMsg, MediaPlayer music) {

    if (a.getText().equals("X") && b.getText().equals("X") && c.getText().equals("X")) {
        // symbolBadge holds the current player's symbol badge (btn1)
        if (symbolBadge.getText().equals("X")) { /* P1 wins */ }
        else { /* P2 wins */ }
    } else if (a.getText().equals("O") && b.getText().equals("O") && c.getText().equals("O")) {
        if (symbolBadge.getText().equals("O")) { /* P1 wins */ }
        else { /* P2 wins */ }
    } else { return false; }
    return true;
}
```

### Board & Turn Management

The 3×3 board is implemented as **9 individual `TextView` cells** (T1–T9) laid out in a `ConstraintLayout` (`lay1`). Each cell is 100×100 dp with alternating dark (`#192043`) and white backgrounds for a visual checkerboard effect.

<div align="center">

| Cell | Position | Background |
|:---:|:---:|:---:|
| T1, T3, T5, T7, T9 | Corners + Centre | `#192043` (Dark Navy) |
| T2, T4, T6, T8 | Edges | `@color/white` |

</div>

**All 8 Winning Combinations Checked:**

```
Rows:       T1-T2-T3 | T4-T5-T6 | T7-T8-T9
Columns:    T1-T4-T7 | T2-T5-T8 | T3-T6-T9
Diagonals:  T1-T5-T9 | T3-T5-T7
```

Every cell's `onClick` fires `cal1()` for each combination that passes through that cell — for example, T5 is checked against all 4 lines (2 diagonals + 1 row + 1 column).

<br/>

### State Management

<div align="center">

| State Variable | Type | Description |
|:---|:---:|:---|
| `isEmpty[]` | String[9] | Tracks which cells are free ("Yes") or taken ("No") |
| `mCount` | int | Running move counter; `mCount % 2` drives turn alternation |
| `cH` | int | Cached initial `mCount` value; used to restore turn order on Reset |
| `isEmptyBoard` | boolean | True until first move — guards against resetting a fresh board |
| `freshlyStarted` | boolean | Ensures `onStart()` initialization runs exactly once |
| `gameHasEnded` | boolean | (PvC only) Prevents further moves after a terminal game state |

</div>

