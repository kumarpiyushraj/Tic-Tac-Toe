<br/>

**Tic-Tac-Toe** is a classic strategy game reimagined for Android. Built entirely in **Java** with XML layouts in Android Studio, this app delivers a polished, feature-rich gaming experience that goes well beyond a basic grid. From a **Lottie-animated splash screen** to a **position-aware AI opponent**, every detail has been thoughtfully crafted.

Whether you're challenging the computer or battling a friend on the same device, the app handles every scenario — victories, draws, score tracking, and mid-game resets — with smooth transitions, custom UI components, and satisfying audio feedback.

<br/>

<div align="center">

| &nbsp; | Why This App? | What It Delivers |
|:---:|:---|:---|
| 🎮 | **Dual Game Modes** | Player vs Computer and Player vs Player, both on a single device |
| 🤖 | **Smart AI Opponent** | Position-aware computer with strategic move prediction and random fallback |
| 🏆 | **Win Line Highlighting** | 8 distinct `ImageView` overlays reveal the exact winning line on every victory |
| 🔊 | **Rich Audio System** | 9 dedicated `MediaPlayer` tracks covering every single interaction in the app |
| 🎨 | **Custom UI Throughout** | Custom Snackbars, Toasts, themed AlertDialogs, PopupMenus, and 10 custom font families |
| ✏️ | **Full Personalization** | Player names (up to 12 chars), symbol choice (X or O), game mode — all configurable |

</div>

### 👥 3. Two-Player (PvP) Mode

`GameBoard.java` drives the full local two-player experience with real-time turn management, score persistence, and an 8-overlay win-line system.

<div align="center">

| Feature | Detail |
|:---|:---|
| 🔄 **Turn System** | `mCount % 2` alternates turns; `tv1` banner updates its background drawable and text on every move |
| 🎨 **Symbol Colors** | **O** renders in blue `#3e59c7`; **X** renders in red `#f2071b` — colour-coded per symbol, not per player |
| 🏅 **Symbol Badges** | `btn1` / `btn2` TextViews beside player names display their symbol with a matching colour background |
| 🖼️ **Player Avatars** | `player` and `robot_player` `ImageView`s switch between `player_red` and `player_blue` drawables |
| 📛 **Marquee Names** | `P1` / `P2` TextViews use `ellipsize=marquee` + `marqueeRepeatLimit=forever` + `singleLine=true` for long names |
| 📊 **Live Score** | `p1s` / `p2s` static `int` counters increment on each win and render as `"Score - N"` in styled badge views |
| 🔄 **Reset Board** | Clears all 9 cells, hides all 8 win-line overlays, re-enables every tile, restores turn to `cH` |
| 🗑️ **Reset Score** | Zeroes both counters — fires custom `msg1` Snackbar instead if both scores are already 0 |
| 🛡️ **Empty Board Guard** | `isEmptyBoard` flag prevents reset on an untouched board — fires `msg2` Snackbar as feedback |
| 📢 **Custom Snackbar** | Two pre-built `Snackbar` instances inflate `custom_snackbar.xml` — transparent BG, styled text, tappable OK |

</div>

```java
// GameBoard.java — Symbol placement with turn tracking
public void setSymbolOnBoard(TextView textview) {
    mVar = (mCount % 2);
    mCount++;
    if (mVar == 0) {
        textview.setText("O");
        tv1.setBackgroundResource(R.drawable.turn_play_red);  // banner = red
        tv1.setText(R.string.x_turn);                         // next: X's turn
        textview.setTextColor(0xFF3e59c7);                    // O = blue
        textview.setEnabled(false);
    } else {
        textview.setText("X");
        tv1.setBackgroundResource(R.drawable.turn_play_blue);
        tv1.setText(R.string.o_turn);
        textview.setTextColor(0xFFf2071b);                    // X = red
        textview.setEnabled(false);
    }
}
```

<a name="4-single-player-pvc-mode--ai-engine"></a>

### 🤖 4. Single-Player (PvC) Mode & AI Engine

`GameBoardSinglePlayer.java` layers a complete AI turn system on top of the shared board. The computer always responds after a **500 ms delay** for a natural game feel.

```mermaid
sequenceDiagram
    participant User
    participant Board as GameBoardSinglePlayer
    participant AI as AI Decision Engine
    participant Handler as Handler (500ms)

    User->>Board: Tap a cell
    Board->>Board: Disable all empty cells (block input)
    Board->>Board: setSymbolOnBoard(cell, byUser=true)
    Board->>Board: isFromUser=false · isComputer=true
    Board->>Board: Check terminal state — cal1() / cal2()

    alt Game not ended
        Board->>AI: isFirstMove? → setSymbolComputerRandom()
        Board->>AI: else → callMethod(lastPos)
        AI-->>Handler: postDelayed 500ms
        Handler->>Board: setSymbolComputer(targetCell)
        Board->>Board: Place symbol · performClick() · re-enable empty cells
        Board->>Board: isFromUser=true · lastPos updated
    end
```

**Two-Phase AI Decision Engine:**

<div align="center">

| Phase | Trigger | Behaviour |
|:---:|:---:|:---|
| **Phase 1 — First Move** | `isFirstMove == true` | Picks a random empty cell via `java.util.Random`; flips `isFirstMove = false` |
| **Phase 2 — Primary** | `callMethod(lastPos)` | Dispatches to the position-specific method for the user's last cell (0–8) |
| **Phase 2 — Fallback A** | Primary returns `null` | Tries alternate line-completion methods for adjacent lines |
| **Phase 2 — Fallback B** | All strategic checks `null` | Falls back to `setSymbolComputerRandom()` to guarantee a valid move |

</div>

```java
// callMethod() — AI dispatch based on the cell the user just played
@Nullable
private TextView callMethod(int x) {
    if (x == 0) return tOne();    // User played T1 → analyse T1's three lines
    else if (x == 1) return tTwo();
    else if (x == 2) return tThree();
    else if (x == 3) return tFour();
    else if (x == 4) return tFive();  // Centre — covers all 4 winning lines
    else if (x == 5) return tSix();
    else if (x == 6) return tSeven();
    else if (x == 7) return tEight();
    else if (x == 8) return tNine();
    else return null;
}
```

`tFive()` illustrates the pattern — it checks every line that passes through the centre cell:

```java
@Nullable
private TextView tFive() {
    if (T5.getText().equals(T1.getText()) && T9.getText().equals("")) return T9;  // diagonal ↘
    if (T5.getText().equals(T9.getText()) && T1.getText().equals("")) return T1;
    if (T5.getText().equals(T3.getText()) && T7.getText().equals("")) return T7;  // diagonal ↙
    if (T5.getText().equals(T7.getText()) && T3.getText().equals("")) return T3;
    if (T5.getText().equals(T4.getText()) && T6.getText().equals("")) return T6;  // row
    if (T5.getText().equals(T6.getText()) && T4.getText().equals("")) return T4;
    if (T5.getText().equals(T2.getText()) && T8.getText().equals("")) return T8;  // column
    if (T5.getText().equals(T8.getText()) && T2.getText().equals("")) return T2;
    return null;
}
```

**PvC State Flags:**

<div align="center">

| Flag | Type | Purpose |
|:---|:---:|:---|
| `isFromUser` | boolean | Blocks cell taps while the AI is thinking |
| `isComputer` | boolean | Gates AI move execution after user input is processed |
| `gameHasEnded` | boolean | Halts both user and AI after any terminal state |
| `isFirstMove` | boolean | Routes the AI's very first response to the random picker |
| `lastPos` | int | Index (0–8) of user's last tap — drives `callMethod()` dispatch |
| `names[]` | TextView[9] | Board cell array enabling index-based AI placement |

</div>

<a name="5-win-detection--draw-system"></a>

### 🏆 5. Win Detection & Draw System

Both `GameBoard` and `GameBoardSinglePlayer` share the same two-method terminal-state engine — `cal1()` for wins and `cal2()` for draws.

```mermaid
graph TD
    A[Cell Tapped] --> B[setSymbolOnBoard]
    B --> C[cal1 — do 3 cells match?]
    C -->|X · X · X| D{symbolBadge = X?}
    C -->|O · O · O| E{symbolBadge = O?}
    D -->|Yes — P1 wins| F[win_red banner · p1s++ · win-line shown · cells off]
    D -->|No — P2 wins| G[win_red banner · p2s++ · win-line shown · cells off]
    E -->|Yes — P1 wins| H[win_blue banner · p1s++ · win-line shown · cells off]
    E -->|No — P2 wins| I[win_blue banner · p2s++ · win-line shown · cells off]
    C -->|No match| J[cal2 — isBoardFilledUp?]
    J -->|All 9 cells taken| K[game_draw banner · turn indicator hidden]
    J -->|Empty cell exists| L[Continue — next player's turn]
```

```java
// isBoardFilledUp() — checks isEmpty[] for any remaining free cell
private boolean isBoardFilledUp() {
    for (int n = 0; n < 9; n++) {
        if (isEmpty[n].equals("Yes")) return false;
    }
    return true;
}
```

> `cal1()` uses the `symbolBadge` (`btn1`) reference — not player number — to decide which score counter to increment. This keeps scoring correctly symbol-aware regardless of which player chose X or O.

### 🔊 Sound & Feedback System

Every interaction in the app is paired with a dedicated audio track. Nine `MediaPlayer` instances are distributed across the full activity lifecycle.

<div align="center">

| Instance | Sound File | Activity | Triggered By |
|:---|:---:|:---:|:---|
| `music` | `button_click` | `First` | Start Game button |
| `music1` | `menu_sound` | `First` | Options / hamburger button |
| `music2` | `radiobutton` | `NameSymbol` | Any RadioButton selection |
| `music3` | `button_click` | `NameSymbol` | Continue & Play Game buttons |
| `music4` | `dialog_msg` | `NameSymbol` | AlertDialog validation triggers |
| `music5` | `button_click` | `GameBoard / SP` | Reset board & Reset score buttons |
| `music6` | `winner` | `GameBoard / SP` | `cal1()` — any winning combination detected |
| `music7` | `game_over` | `GameBoard / SP` | `cal2()` — board full with no winner |
| `music8` | `board_click` | `GameBoard / SP` | Every cell tap on the game board |

</div>

> On a board reset, if `music6` (winner) or `music7` (game_over) is still playing, it is explicitly `stop()`ed and `prepare()`d so it can be reused in the next round without allocating a new instance.

### State Variables

<div align="center">

| Variable | Type | Description |

### Build from Source

```bash
# 1. Clone the repository
git clone https://github.com/kumarpiyushraj/Tic-Tac-Toe.git
cd Tic-Tac-Toe

# 2. Open in Android Studio
#    File > Open > navigate to the cloned folder > OK

# 3. Gradle sync
#    Click "Sync Now" when prompted — all dependencies resolve automatically

# 4. Verify SDK
#    Tools > SDK Manager > ensure API 21+ is installed

# 5. Run
#    Connect a USB-debugging device  OR  launch an AVD from AVD Manager
#    Press Run ▶️  or  Shift + F10
```

**End-User Requirements** — No special permissions needed. The app works fully offline.
